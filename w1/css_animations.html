Animations Using CSS

There is a CSS property animation. It has several sub-properties:

  • animation-name
  • animation-duration
  • animation-timing-function
  • animation-delay
  • animation-iteration-count
  • animation-direction
  • animation-fill-mode
  • animatin-play-state

This example also makes use of another sub-propert: animation-fill-mode. It is set to the valueboth as a quick way to keep the style of the animation. Since the height starts at 0, without the animation-fill-mode: both;, then after the animation the height would jump back to 0. This can be seen in first Adding Directions example. After the circle moves to the right, it jumps back to the left. Truthfully, the reason the very first example (Simple Keyframe Adjustments) does not do this is because I didn't know there was a property at the time of writing. So I purposefully made it start on the right and move left. The CSS for the circle defaults to the left. The animation from is what kicked it to the right at the very beginning. After the animation ends, it jumps back to the default CSS for the circle because I did not set the animation-fill-mode sub-property.

These keyword values for the property are all presets of the cubic-bezier(p1, p2, p3, p4) function. So you can always use animation-fill-mode: cubic-bezier(a, b, c, d);.

Did Someone Say Pause?

  • paused
  • running

The animation-play-state sub-property sets whether an animation is running or paused. The example below has the inside-box being animated. It also sets animation-play-state: paused; on hover for the inside box .inside-box:hover { animation-play-state: paused; }.
