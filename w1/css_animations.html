Week 1

Animations Using CSS

There is a CSS property animation. It has several sub-properties:

  • animation-name
  • animation-duration
  • animation-timing-function
  • animation-delay
  • animation-iteration-count
  • animation-direction
  • animation-fill-mode
  • animatin-play-state

Examples

Simple Keyframe Adjustments

Here is an element that has an animation that uses @keyframes to animate the CSS left attribute from the 100% - 225px (the difference to prevent overflow) to 0. It happens over 3 seconds which is set with the CSS animation-duration attribute.

You may need to refresh the browser to see this animation, as it only plays once.

/* Let's build something visual */
.circle {
  width: 200px;
  height: 200px;
  background-color: blue;
  border: .5em solid #666;
  border-radius: 135px;
  position: relative;
}

/* Now... let's animate */
#simpleSlide {
  animation-name: slideLeft; /* Referenced by @keyframe */
  animation-duration: 3s; /* How long the animation takes for 1 cycle */
}

@keyframes slideLeft {
  from {
    left: calc(100% - 225px);
  }
  to {
    left: 0;
  }
}

This example also makes use of another sub-propert: animation-fill-mode. It is set to the valueboth as a quick way to keep the style of the animation. Since the height starts at 0, without the animation-fill-mode: both;, then after the animation the height would jump back to 0. This can be seen in first Adding Directions example. After the circle moves to the right, it jumps back to the left. Truthfully, the reason the very first example (Simple Keyframe Adjustments) does not do this is because I didn't know there was a property at the time of writing. So I purposefully made it start on the right and move left. The CSS for the circle defaults to the left. The animation from is what kicked it to the right at the very beginning. After the animation ends, it jumps back to the default CSS for the circle because I did not set the animation-fill-mode sub-property.

These keyword values for the property are all presets of the cubic-bezier(p1, p2, p3, p4) function. So you can always use animation-fill-mode: cubic-bezier(a, b, c, d);.

Do We Need To Pause A Minute?

  • paused
  • running

The animation-play-state sub-property sets whether an animation is running or paused. The example below has the inside-box being animated. It also sets animation-play-state: paused; on hover for the inside box .inside-box:hover { animation-play-state: paused; }. When you remove the curser from the inside box (blue box) then the animation resumes.

@Keyframes

Keyframes use a percentage value to indicate the time during an animation sequence. It turns out, that the from and to keywords are really just aliases for 0% and 100% respectively. You can even add several additional keyframes to describe steps between the start and end of the animation. The example below builds off the previous example where the width is moving from 0% to 100%, but I am adding additional keyframes every 25% to change the background color. Now the inside box will change colors as it animates.

.inside-box-2 {
  width: 50%;
  height: 250px;

  animation-name: moving-box-2;
  animation-duration: 4s;
  animation-iteration-count: infinite;
  animation-direction: alternate;
  animation-timing-function: ease-in-out;
}

@keyframes moving-box-2 {
  from {
    width: 0;
  }

  25% {
    background-color: red;
  }

  50% {
    background-color: orange;
  }

  75% {
    background-color: yellow;
  }

  100% {
    width: 100%;
    background-color: green;
  }
}

Trying Something Fun

This little loading sequence utilizes one animation (load-horizontally). There are 7 dots, each with a unique class (although an ID might have been more appropriate). The unique class references the animation and sets a different animation-delay based on where the dot falls in the order. This animation is also making use of the steps function for the animation-timing-function: animation-timing-function: steps(100, end);. This means there will be 100 evenly divided keyframes over the course of the animation-duration, which in this example is set to 10 seconds.

There is only one keyframe defined for this example, 100%. I do not specify the 0%, so the default CSS is used to start with. The base class loading-dot defined the shape of the dot and set the positions to absolute so all the dots were perfectly on top of each other. This prevented the need to specify a left: # px; for each unique dot class.
